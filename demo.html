1.git 将代码分布管理

1.Git 记录每一次章节的变动 1.git init 2.git add 3.git commit -m '' 2.git reset HEAD 撤销上一步 3.git checkout -- file // 会将上一次提交暂存区的文件 覆盖本次修改的文件 4.git log // 提交历史

1.回到过去 reset checkout reset 回滚快照 HEAD~ 一个~表示回滚一次 ~~表示回滚两次 or ~10 回滚十次 .git reset soft HEAD~ 移动HEAD的指向，将其指向上一个快照 相当于撤回一次错误的commit .git rest --mixed HEAD~ H mixed默认 移动HEAD的指向，将其指向上一个快照 将移动后指向的快照滚回到暂存区 .git reset --hard HEAD~ 移动HEAD的指向，将其指向上一个快照 将移动后指向的快照滚回到暂存区 将暂存区的文件还原到工作目录 回滚快照里面个别文件 .git reset 版本快照 文件名/路径 比较工作区与暂存区的目录 比较两个历史快照 git diff 快照ID1 快照ID2 比较工作目录和git 仓库中的快照 git diff 快照ID 比较工作目录和git 仓库中的最新提交的快照 git diff HEAD
